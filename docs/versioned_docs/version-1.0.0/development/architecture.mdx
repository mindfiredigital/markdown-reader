# Architecture

Markdown Reader has two main parts: the desktop main process and the React renderer.

## Main process

The main process manages system-level work:

- native windows
- file open dialogs
- file reading
- file watching
- native menus
- recent files
- export handlers
- OS integration

## Renderer

The renderer is the user interface:

- document viewer
- table of contents sidebar
- file browser
- tabs
- search UI
- welcome screen
- settings controls
- rendered Markdown output

## Markdown pipeline

A typical rendering flow:

1. User opens a Markdown file.
2. Main process reads the file content.
3. Renderer parses Markdown.
4. HTML is sanitized before reaching the DOM.
5. Rich content such as code, diagrams, and math is rendered.
6. TOC is generated from headings.

## Runtime strategy

The app can use Electron for stable cross-platform desktop builds. The long-term architecture keeps the renderer reusable so desktop runtime changes affect only the platform adapter layer.
